## [0.6.0] - 2026-07-22

### 去工具化 + 说人话 + 群友更名 + 主持人机制 + 格式统一 + 来源归档

#### 变更

##### 1. 去工具化，说人话
- **去除工具性名词**：正文中不再出现"JTBD""方法论引擎""方法论工具箱"等术语
  - "方法论" → "思考方式"
  - "JTBD" → "用户任务法"
  - "方法论库" → "思考方式库"
  - "跨学科框架扫描" → "跨学科扫描"
  - "层次衔接检验" → "互相找茬"
- **说人话**：用生活类比解释复杂概念，让六年级小学生都能听懂
  - "锤子钉子效应" → "拿着锤子找钉子"
  - "变量/常数" → "会变的/不变的"
  - "正文中不提思考方式的名称" → "不要说'我用XX思维来分析'"
- **字数缩减**：标准输出从1000字/人 → **500字/人**
  - 高复杂度：1000字+/人 → 600-700字/人
  - 续接追问：800字 → 400字
  - 续接扩展/自定义：全员500字 → 全员300字

##### 2. 群友更名
- 老A → **老齐**
- B哥 → **方鸣**
- C姐 → **沈锐**
- D弟 → **苏慎**
- SKILL.md 和 roles/SKILL.md 全局同步替换，零残留

##### 3. 主持人机制
- **新增"主持人"角色**：每轮讨论由主持人开场和串联
  - 第一位群友发言前：主持人直接抛出话题（如"针对XX，老齐你怎么看？"）
  - 后续群友发言前：主持人一句话总结前一人 + 引导下一人
  - **每次铺垫措辞必须不同**，像组局的朋友，轻松自然
- **互怼环节主持人引导**：有时主持人挑起话题，有时让群友自然碰撞，交替使用
- **续接轮次也加主持人铺垫**：比首轮更轻量，一句话带过

##### 4. 发言格式统一
- 群友发言的**每一个自然段落**都必须以 `[名字]：` 开头
- 主持人说话的**每一个自然段落**也必须以 `[主持人]：` 开头
- 互怼发言格式：`[老齐]：方鸣，你说的那个思路有一个前提没验证——`
- 续接轮次同样适用

##### 5. 口头禅灵活化
- 从"固定台词"改为"风格描述"
- 每个角色保留风格倾向（如老齐"先拆本质"、苏慎"先叫停挑漏洞"），但**每次措辞不同**
- 不再列固定句子让AI照着说，而是描述风格让AI自由发挥

##### 6. 数据来源归档
- **正文中不出现任何引用标记、链接或脚注**
- 所有数据来源集中归档到文末"参考资料"区域，按出现顺序列出
- 保持主体内容的阅读流畅性

#### 涉及文件
- `SKILL.md` — 主控文件全面更新
- `roles/SKILL.md` — 角色定义同步更新
- `README.md` — 说明文档全面更新
- `CHANGELOG.md` — 新增v0.6.0记录

#### 设计决策
- "去工具化"不是去掉分析框架，而是把框架变成底层思考逻辑自然融入叙述，不让读者感到"在被教学"
- 主持人不是"央视主持人"的正式腔，是"组局朋友"的轻松腔，不喧宾夺主
- 口头禅灵活化是为了避免机械重复，让每次讨论都有新鲜感
- 来源归档是为了让主体内容纯净阅读，同时保留数据可溯源性

## [0.5.0] - 2026-07-19

### 新增续接机制：支持连续对话，首轮与续接双模式并行

#### 新增
- **Step 7 续接机制**：Step 6 汇总后弹出续接选择，用户可选追问/扩展/换角度/自定义/新话题/收尾
- **7.1 续接选项**（6项）：追问（指定群友深入）、扩展（某维度展开）、换角度（新视角）、自定义（自由输入）、新话题（重置上下文开新局）、收尾（结束）
- **7.2 续接轮次轻量规则**：跳过Step 0、搜索减为1组精准补充、发言字数降低（追问800字/扩展500字）、互怼可选
- **7.3 续接轮次发言风格**：保留"老A："开头和由浅入深结构，但不要求固定"核心观点"开头和"怎么办"结尾，群友基于上下文自然接话
- **7.4 上下文摘要机制（内部工作记忆）**：每轮结束后系统内部生成摘要（不对外展示），含群友核心观点、争议点、追问方向、方法论清单，下一轮基于摘要自然接话
- **7.5 新话题重置**：选择"新话题"时自动清空上下文摘要，重跑完整Step 0-6，确保不同话题之间无信息干扰
- **7.6 循环机制**：续接轮次结束后再次弹出续接选择，直到用户选"收尾"才结束
- **输出格式模板追加续接选择提示**：Step 6 之后新增续接选项引导

#### 设计决策
- 不破坏现有Step 0-6的主体结构，续接机制作为Step 7独立追加
- 首轮走完整流程（一次性分析/内容产出），续接轮走轻量流程（连续探讨/深度迭代）
- 上下文摘要只在系统内部流转，不增加用户认知负担
- "新话题"带完整重置，解决同一对话中多话题分析的信息干扰问题

## [0.4.1] - 2026-07-18

### 总管新增跨学科框架扫描，防止"锤子钉子"效应

#### 新增
- **Step 0.3 从"方法论预匹配"升级为两阶段流程**：
  - **第一阶段：跨学科框架扫描**——在从方法论库里选工具之前，先问"这个问题本身最适合用什么分析框架"
  - **话题降维**：把话题拆解为最本质的几个问题
  - **跨学科映射**：扫描经济学/心理学/社会学/传播学/法学/系统工程等学科视角
  - **框架缺口检测**：方法论库里没有对应工具时，用系统性思考补充（不因工具缺失而遗漏视角）
  - **反锤子检测**：检查预选方法论是否因为"有这个工具所以用"而非"最适合用这个"
  - **第二阶段：方法论覆盖预匹配**——基于第一阶段结论，从方法论库中匹配

#### 设计决策
- 核心理念：先确定"应该用什么框架"，再从工具箱里找工具，而不是倒过来
- 跨学科扫描结果可能发现方法论库里没有对应skill的维度——此时用系统性思考补充，而非跳过该维度
- 搜索中如果发现新的分析框架比现有方法论更合适，允许在分析中自然融入

## [0.4.0] - 2026-07-18

### 内嵌总管：Step 0 升级为问题预处理系统

#### 新增
- **Step 0 从"简单诊断"升级为"深度预处理"**——把总管能力嵌入圆桌流程，用户感知不变（一句话启动），系统后台自动执行高质量预处理
- **0.1 结构化分析**：话题维度拆解、隐含前提识别、目标受众推断
- **0.2 智能追问**：只在问题模糊时追问，追问方式是具体问题而非"能补充一下吗"
- **0.3 方法论覆盖预匹配**：搜索前预判需要哪些方法论，主维度2-3个+交叉维度1-2个+盲区自检
- **0.4 输出规格适配**：根据问题难度动态调整输出规格（简单200-400字/人 → 标准1000字/人 → 深度1000字+/人）
- **0.5 输出质量预检规则**：分析完成后自检论据来源、数据准确性、方法论矛盾、字数达标、盲区覆盖
- **Step 1 增强**：新增来源质量预检（优先权威来源）和时效性判断（超2年需标注）
- **Step 2 增强**：基于0.3预匹配+搜索结果联合选择，新增动态调整和盲区补漏机制

#### 设计决策
- 总管功能嵌入Step 0而非独立skill——用户体验更丝滑，避免两步触发
- 追问时机为"问题模糊时"而非"每次分析前"——不浪费用户时间
- 预匹配在搜索前执行、正式选择在搜索后执行——两层过滤确保方法论覆盖完整

## [0.2.0] - 2026-07-18

### 角色系统重构 + 输出风格大改

#### 变更
- **去除固定角色人设**：不再使用"A看山/B修路/C织网/D照镜/E种树"的固定人物设定
  - 原因：AI模仿具体人设容易"学虎不成反类犬"，显得做作生硬
- **改为分析倾向驱动**：4个"群友"只有分析关注点（底层/方法论/战术/批判），不演人
  - 老A：底层/本质/趋势
  - B哥：方法论/框架
  - C姐：战术/实操
  - D弟：批判/风险/工具
- **去掉第5个角色**（原E种树·人文关怀师）：人文关怀可以在任何群友的发言中自然融入
- **输出风格从"咨询报告"改为"B站知识区群聊"**：
  - 犀利但不装逼
  - 偶尔抖机灵但不刻意
  - 商业案例和生活类比交替
  - 绝不说教
  - 禁止：教授腔、MBA腔、鸡汤腔、网红腔
- **输出模板全面改版**：
  - "第一轮：独立分析" → "开聊"
  - "交叉讨论" → "互怼环节"
  - "共识点" → "大家都同意的"
  - "分歧点" → "吵得最凶的"
  - "可执行建议" → "所以到底该怎么办"
- **新增方法论动态迭代规则**：
  - 一轮讨论中群友不变动
  - 方法论可以因新信息出现而迭代、更新、更换
  - 更换时在讨论中自然说明原因

#### 目标用户明确
- 刚毕业1-5年的年轻人
- 不缺理论，缺怎么用和踩坑前的预判
- 讨厌说教，喜欢一针见血的金句
- 语境是B站/小红书/即刻/播客，不是MBA课堂
