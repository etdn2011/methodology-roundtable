# 非暴力沟通

## 思想来源
源自马歇尔·卢森堡(Marshall Rosenberg)《非暴力沟通》(2003)。

## 一句话定义
大部分冲突的根源不是"谁对谁错"，而是双方都没有表达出真正的需求和感受。非暴力沟通提供了一套"不带评判"的表达和倾听方式，让对方听到你的需求，同时你也听到对方的。

## 核心逻辑

### 暴力沟通 vs 非暴力沟通
- 暴力沟通：评判 + 比较 + 命令 + 回避责任
  - "你总是这样"（评判）
  - "你看看别人家老公"（比较）
  - "你必须听我的"（命令）
  - "这不是我的错"（回避责任）
- 非暴力沟通：观察 + 感受 + 需求 + 请求
  - "我看到你昨晚10点才回家"（观察，不带评判）
  - "我感到担心"（感受，不带指责）
  - "因为我需要一个确定感"（需求）
  - "以后能不能提前告诉我？"（请求，不是命令）

### 四个步骤

1. **观察（而非评判）**：说出你看到的事实，不加评价。"你迟到了"是评判，"你比约定时间晚到了30分钟"是观察。
2. **感受（而非想法）**：说出你的情绪，而不是你的判断。"我觉得你不尊重我"是想法，"我感到失望和生气"是感受。
3. **需求（而非策略）**：说出你为什么有这个感受。"我需要被重视和尊重"是需求，"你必须准时"是策略。
4. **请求（而非命令）**：提出你希望对方做什么，而不是命令。"你能每周至少有一天准时回家吃饭吗？"是请求。

### 关键洞察
- 大部分人不会直接说出需求——他们会用愤怒、沉默、被动攻击来表达
- 倾听的核心不是"回应对方的话"，而是"听到对方的话背后的需求和感受"
- 非暴力沟通不是"没有冲突"——而是让冲突变得有建设性

## 操作步骤

### 步骤1：表达前先自我校准
在开口之前，问自己：
- 我接下来要说的话是观察还是评判？
- 我是在说感受还是在说指责？
- 我知道自己的需求是什么吗？

### 步骤2：用四步框架表达
按照"观察→感受→需求→请求"的结构表达自己。

### 步骤3：倾听对方
当对方说话时：
- 不要打断
- 不要急着反驳
- 尝试听到对方话语背后的"感受"和"需求"——"你总是控制我"背后的需求可能是"我需要自主权"

### 步骤4：复述确认
"我听到你说……你的感受是……你需要的是……对吗？"——确认你理解对了。

### 步骤5：共同寻找解决方案
当双方的需求都被表达和听到后，一起找满足双方需求的方案。

## 参考案例
卢森堡在书中记录了一个经典场景：一个幼儿园老师对他说"有些孩子简直不可救药"。卢森堡没有用评判回应，而是用非暴力沟通的方式问："哪些具体行为让你感到困扰？你希望他们怎么做？"通过这种不带评判的提问，老师开始表达自己的感受和需求——她不是讨厌这些孩子，她是感到无力、需要支持和新的方法。一旦需求被表达出来，解决方案就变得清晰了。另一个案例是：卢森堡在巴勒斯坦和以色列的冲突地区进行非暴力沟通工作坊，让敌对双方用NVC的四步框架互相表达——不是"你们做了什么"，而是"我经历了什么、我的感受是什么、我的需求是什么"。结果参与者第一次"听到"了对方的人性。

## 适用条件
- 特别有效：高情绪对话、亲密关系冲突、亲子沟通、职场矛盾调解
- 需谨慎：紧急决策场景（着火时不需要NVC，直接行动）
- 可能误导：用NVC来"套路"对方——非暴力沟通的核心是真诚，不是技巧

## 局限性
- NVC需要双方都有意愿——如果一方完全不愿意沟通，NVC也无效
- 在权力严重不对等的关系中，"请求"可能变成"命令"
- 文化差异：在某些文化中，直接表达感受和需求本身就被视为不礼貌

## 相关方法论
- 用户任务法关系版（理解对方的深层需求是NVC的基础）
- 绝对坦率（绝对坦率解决"敢不敢说"，NVC解决"怎么说"）
- 灰度管理（NVC本身需要在"直接"和"委婉"之间找到灰度）
- 影响力六原则（NVC中使用"喜好"和"互惠"原则来建立信任）
