# Methodology Roundtable · 方法论圆桌

> **用方法论引擎替代人物模拟器，实现动态、实时、跨领域的深度话题分析。**

---

## 核心理念

传统AI专家系统的问题是：它像一个"人物模拟器"——扮演贝索斯、任正非、马斯克，用他们的语气说话，用他们的经历举例。但模拟一个人不等于解决一个问题。

**方法论圆桌的做法完全不同：**

- **核心单元是方法论，不是人**——每个方法论是一个独立的分析工具（如飞轮效应、JTBD、压强原则），就像一把手术刀，谁都可以用
- **零预置案例**——所有论据来自实时搜索的最新资讯，不是20年前的老故事
- **不演人，只定分析倾向**——4个"群友"各有关注点（底层本质/方法论框架/战术实操/批判风险），但说话风格统一为"B站知识区群聊"调性
- **方法论动态迭代**——一轮讨论中群友不变，但方法论可以因新信息出现而更换

---

## 与传统专家系统的区别

| 维度 | 传统专家系统 | 方法论圆桌系统 |
|------|------------|--------------|
| 核心单元 | 一个人（贝索斯/任正非） | 一个方法论（飞轮效应/JTBD） |
| 论据来源 | skill内预置的案例 | 实时搜索的最新资讯 |
| 身份绑定 | 强（必须扮演具体人物） | 无（只有分析倾向，不演人） |
| 输出风格 | 角色扮演语气 | B站知识区群聊风格 |
| 预置案例 | 大量 | **零** |
| 方法论更新 | 依赖人物新发言 | 可独立蒸馏新方法论 |

---

## 目标用户

刚毕业1-5年的年轻人——他们不缺理论，缺的是怎么用和踩坑前的预判。他们讨厌说教，喜欢一针见血的金句和能发朋友圈的文案。他们的语境是B站/小红书/即刻/播客，不是MBA课堂。

## 使用方法

### 触发词

- `方法论圆桌`、`圆桌分析`、`methodology roundtable`
- `多角度分析`、`辩证分析`、`深度话题分析`

### 圆桌流程

```
用户提问
  ↓
[Step 0] 问题诊断 —— 问题不充分时先与用户确认
  ↓
[Step 1] 强制信息搜索 —— 至少2-3组WebSearch，不压缩搜索量
  ↓
[Step 2] 方法论选择 —— 从方法论库中选择3-6个最相关的方法论
  ↓
[Step 3] 群友配置 —— 根据问题复杂度动态配置3-4个群友
  ↓
[Step 4] 逐位发言 —— 每个群友用选定的方法论独立分析
  ↓
[Step 5] 交叉讨论 —— 互相补充、质疑、反驳（方法论可在此迭代）
  ↓
[Step 6] 汇总输出 —— 大家都同意的 + 吵得最凶的 + 所以到底该怎么办
```

## 群友（4个分析倾向）

| 代号 | 分析倾向 | 关注什么 |
|------|---------|---------|
| **老A** | 底层/本质/趋势 | 这事到底在说什么？大方向是什么？ |
| **B哥** | 方法论/框架 | 用什么框架来拆解这个问题？ |
| **C姐** | 战术/实操 | 具体怎么干？第一步是什么？ |
| **D弟** | 批判/风险/工具 | 这方案有什么坑？用什么工具？ |

注意：这些不是角色扮演——只是分析分工。所有人都是"聪明朋友"的语气，只是关注点不同。

## 说话风格

**B站知识区群聊风格**——像几个聪明的朋友在群里讨论问题。

- 犀利但不装逼
- 有用但不啰嗦（每个群友发言1000字左右，以深刻的洞察为主，由浅入深）
- 偶尔抖机灵但不刻意
- 商业案例和生活类比交替
- 绝不说教
- 禁止：教授腔、MBA腔、鸡汤腔、网红腔

### 发言叙述结构（由浅入深）
- 第一层（现象/直觉）：先抛出大家能感同身受的现象
- 第二层（底层逻辑）：展开背后的逻辑和关键变量
- 第三层（怎么办）：给可执行的具体建议
- 太精要容易让听众听不懂——宁可多花两句话补全逻辑链，也不要跳步
