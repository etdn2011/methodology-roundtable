# Methodology Roundtable · 方法论圆桌

> **用方法论引擎替代人物模拟器，实现动态、实时、跨领域的深度话题分析。**

---

## 核心理念

传统AI专家系统的问题是：它像一个"人物模拟器"——扮演贝索斯、任正非、马斯克，用他们的语气说话，用他们的经历举例。但模拟一个人不等于解决一个问题。

**方法论圆桌的做法完全不同：**

- **核心单元是方法论，不是人**——每个方法论是一个独立的分析工具（如飞轮效应、JTBD、压强原则），就像一把手术刀，谁都可以用
- **零预置案例**——所有论据来自实时搜索的最新资讯，不是20年前的老故事
- **零人物绑定**——方法论标注思想来源，但不扮演任何人。能抓到老鼠就是好猫
- **方法论+角色双驱动**——方法论提供分析工具，角色（A看山、B修路等）提供分析视角和人味

---

## 与传统专家系统的区别

| 维度 | 传统专家系统 | 方法论圆桌系统 |
|------|------------|--------------|
| 核心单元 | 一个人（贝索斯/任正非） | 一个方法论（飞轮效应/JTBD） |
| 论据来源 | skill内预置的案例 | 实时搜索的最新资讯 |
| 身份绑定 | 强（必须扮演具体人物） | 无（方法论可用在任何领域） |
| 输出风格 | 角色扮演语气 | 分析师语气，有角色化人味 |
| 预置案例 | 大量 | **零** |
| 方法论更新 | 依赖人物新发言 | 可独立蒸馏新方法论 |

## 使用方法

### 触发词

当用户消息包含以下触发词时，系统自动激活方法论圆桌：

- `方法论圆桌`、`圆桌分析`、`methodology roundtable`
- `多角度分析`、`辩证分析`、`深度话题分析`
- `帮我从多个角度分析`、`专家圆桌`

### 圆桌流程

```
用户提问
  ↓
[Step 0] 问题诊断 —— 问题不充分时先与用户确认
  ↓
[Step 1] 强制信息搜索 —— 至少2-3组WebSearch，不压缩搜索量
  ↓
[Step 2] 方法论选择 —— 从方法论库中选择3-6个最相关的方法论
  ↓
[Step 3] 角色配置 —— 根据问题复杂度动态配置3-5个角色
  ↓
[Step 4] 逐位发言 —— 每个角色用选定的方法论独立分析
  ↓
[Step 5] 交叉讨论 —— 互相补充、质疑、反驳
  ↓
[Step 6] 汇总输出 —— 共识 + 分歧 + 可执行建议 + 角色意见矩阵
```

## 设计原则

1. **零预置案例**——所有举例通过 WebSearch 实时获取
2. **零人物绑定**——只标注思想来源，不扮演任何人
3. **操作步骤化**——每个方法论像一份"使用说明书"
4. **说给普通人听**——避免术语堆砌，用类比和日常语言
5. **用户确认前置**——问题不充分时不猜测，先和用户确认
